# Purpose

Define the definitive design constraints for Blueprint.

---

# 1. Blueprint SHALL NOT

| Constraint | Reason |
|-----------|--------|
| Invent architectural decisions | All decisions are frozen in Knowledge Base |
| Introduce new terminology | All terminology is frozen in Glossary |
| Redefine canonical concepts | All concepts are frozen in Foundation |
| Duplicate Foundation | Foundation is already canonical |
| Duplicate existing specifications | TJS-020, TJS-021, TJS-022 are already canonical |
| Change Knowledge IDs | All KB IDs are frozen |

---

# 2. Blueprint SHALL ONLY

| Permission | Description |
|-----------|-------------|
| Organize | Arrange Knowledge Base items into specification sections |
| Structure | Define section hierarchy and ordering |
| Compose | Combine Knowledge Items into normative statements |
| Trace | Map every statement to its Knowledge ID |
| Document | Write the specification using frozen knowledge |
