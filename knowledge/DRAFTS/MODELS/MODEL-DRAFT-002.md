# MODEL-DRAFT-002_Publication_As_Information_Object

**Document ID:** MODEL-DRAFT-002

**Title:** Publication as Information Object

**Status:** Draft

---

# Model Description

Publication IS an Information Object created specifically for Communication purpose. It is Information that has been given identity, lifecycle, and purpose.

---

# Advantages

- Explains why Publication has identity (Territory + Date + Template)
- Explains why Publication has lifecycle (Generated → Archived)
- Explains why Publication has ownership (Publication Engine)
- Explains why Publication bridges Information and Representation
- Explains why Publication is central to the ontology

---

# Weaknesses

- "Information Object" is not a standard term
- May be too abstract for some audiences
- Does not explain territorial grammar
- Does not explain deterministic rendering

---

# Evidence

**Supporting:**

- CASE-COM-001: "Publication is an Information Object created for Communication purpose"
- CASE-INF-001: "Publication is Information structured for distribution"
- A-6.7: "Publication appears BETWEEN Information and Representation"
- CASE-001B: "Publication packages Information for delivery"

**Contradicting:**

- None significant

---

# Rejected Assumptions

- Publication is just Information (rejected: Publication has identity, lifecycle, ownership)
- Publication is just Communication (rejected: Publication is INPUT to Communication)
- Publication is independent of Information (rejected: Publication contains Information)
