# 1. Purpose

This document establishes the scope, context and research questions for CASE-DOCARCH-001: Documentation Architecture.

---

# 2. Case Scope

## 2.1 What This Case Investigates

The DOCUMENTATION ARCHITECTURE for the Publisher repository.

NOT content design.

ONLY documentation architecture.

## 2.2 What This Case Does NOT Investigate

- Document content design
- Specification writing
- Implementation design
- Architecture decisions

## 2.3 Investigation Constraint

Publisher is the reference subsystem.

Telegram Publisher and Facebook Publisher are the first two concrete channel adapters.

We are NOT designing the entire SvitloSk architecture.

We are designing the ideal documentation repository for Publisher.

---

# 3. Context

## 3.1 Previous Investigations

All previous CASE investigations provide knowledge base for this investigation.

## 3.2 The Goal

Determine:

1. Which documents MUST exist in the Publisher repository
2. Which documents are missing
3. Which documents should become normative
4. Which documents should remain informative
5. Which documents belong to Publisher Core, Telegram Publisher, Facebook Publisher, or Shared Publishing Domain
6. Which knowledge should remain outside Publisher

## 3.3 The Constraint

Do NOT create canonical documents.

Do NOT move documents.

Do NOT modify the Publisher Knowledge Base.

This investigation prepares the blueprint for future specifications.
