# 1. Purpose

---

# 2. Case Scope

## 2.1 What This Case Investigates

The evolution of Publisher knowledge across the history of the SvitloSk project.

NOT software evolution.

NOT implementation evolution.

ONLY knowledge evolution.

## 2.2 What This Case Does NOT Investigate

- Software evolution
- Implementation evolution
- New architecture design
- Specification modification

## 2.3 Investigation Constraint

Study every archived project version, baseline, historical document and draft that may contain Publisher concepts.

Do NOT limit yourself to current specifications.

Research the history.

---

# 3. Context

## 3.1 Historical Sources

The project has archived materials in:

- archive/baselines/ (project baselines)
- archive/knowledge/ (historical knowledge)
- archive/phase2/ (phase 2 artifacts)
- archive/working/ (working materials)
- telegram/legacy/ (legacy specifications)
- telegram/investigation/ (investigation artifacts)

## 3.2 The Goal

Reconstruct KNOWLEDGE EVOLUTION.

For every archived version determine:

- Which Publisher concepts existed
- Which concepts disappeared later
- Which concepts evolved
- Which concepts became more precise
- Which concepts were abandoned
- Which concepts still look architecturally correct
- Which concepts contradict newer versions
- Which concepts should be considered historical only
