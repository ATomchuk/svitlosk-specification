# 1. Purpose

# 2. Case Scope

## 2.1 What This Case Investigates

The META-MODEL used for classification.

NOT Publisher.

NOT Journal.

NOT Publication.

ONLY the classification categories themselves.

## 2.2 What This Case Does NOT Investigate

- Classifying project entities
- Redesigning architecture
- Generating documentation
- Renaming concepts

## 2.3 Investigation Constraint

Before classifying entities we must validate the META-MODEL used for classification.

This investigation validates the ontology language itself.

---

# 3. Context

## 3.1 Previous Investigations

Previous investigations showed that entity classification is inconsistent because the meta-model itself has never been validated.

## 3.2 The Goal

Validate the meaning of the classification categories.

For every category determine:

1. Formal definition
2. Necessary properties
3. Sufficient properties
4. Can identity exist?
5. Can lifecycle exist?
6. Can multiple instances exist?
7. Can versions exist?
8. Can ownership exist?
9. Examples from software engineering
10. Examples from information systems
11. Examples from our project
12. Common mistakes
13. Typical confusion with neighbouring categories

## 3.3 The Constraint

This investigation validates the ontology language itself.

It does NOT classify project entities.

It does NOT redesign architecture.

It does NOT generate documentation.

Only evidence.
