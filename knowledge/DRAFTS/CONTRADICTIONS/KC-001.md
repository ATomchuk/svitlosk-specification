**Title:** Publication Definition Inconsistency

**Status:** Draft

---

# Origin

CASE-001, CASE-001A, CASE-COM-001

---

# Description

Publication is defined differently across multiple documents:

- GLOSSARY.md: "A public information message generated by the SvitloSk system"
- TELEGRAM_GLOSSARY.md: "One independent publication belonging to an Issue"
- DATA_MODEL.md: "Logical representation of interpreted information prepared for public distribution"
- TELEGRAM_SEMANTIC_MODEL.md: "Publication represents one independent publication belonging to an Issue"

---

# Affected Concepts

- Publication
- Information
- Message
- Representation

---

# Current Understanding

Each definition emphasizes different aspects:

- "message" — transmission aspect
- "unit" — structural aspect
- "representation" — data model aspect

These are not contradictory but complementary views.

---

# Possible Resolutions

1. Unify definitions into single canonical definition
2. Accept multiple definitions as different views (projection model)
3. Create explicit definition hierarchy
