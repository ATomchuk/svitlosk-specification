**Document ID:** KC-005

**Title:** Domain and Architecture Separation vs Mixed Concepts

**Status:** Draft

---

# Origin

A-6.6, META-002

---

# Description

Investigations found that Domain Kernel and Architecture Kernel have ZERO overlap (A-6.6). But some concepts appear to mix both (e.g., Parser is both Object and Process).

---

# Affected Concepts

- Domain Kernel
- Architecture Kernel
- Parser
- Publication Engine
- Component
- Role

---

# Current Understanding

- Domain and Architecture are ontologically separate
- But some concepts exist at multiple meta-levels
- Parser is Domain Object (business concept) AND Architectural Component (implementation)
- This is not contradiction but multi-level existence

---

# Possible Resolutions

1. Accept multi-level existence as normal
2. Create explicit separation between Domain and Architectural views
3. Redefine concepts to be purely Domain or purely Architectural
