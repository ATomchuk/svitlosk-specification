# 1. Purpose

This document analyzes the ontological categories of each concept inside the Journal, determining what actually exists, what is a representation, what is a process, and what is an implementation detail.

# 2. Concept Analysis

## 2.1 Journal

### Evidence

> "The SvitloSk Public Information Journal is the primary public communication channel of the project." (CHARTER.md §10)

> "It serves as a public archive of processed information, daily publications, analytical summaries and official project communications." (CHARTER.md §10)

### Ontological Category

**Journal is NOT a single category.**

Evidence suggests Journal is simultaneously:

1. **Service** — "public information service" (CHARTER.md §1)
2. **Channel** — "primary public communication channel" (CHARTER.md §10)
3. **Archive** — "public archive of processed information" (CHARTER.md §10)

### Analysis

Journal is a POLYMORPHIC concept:
- As a SERVICE, it analyzes, interprets and distributes information
- As a CHANNEL, it delivers information to residents
- As an ARCHIVE, it preserves historical information

### Conclusion

Journal is a POLYMORPHIC CONCEPT that cannot be reduced to a single ontological category.

---

## 2.2 Publication

### Evidence

> "Publication: A public information message generated by the SvitloSk system for distribution through one or more supported communication channels." (GLOSSARY.md §3)

> "Every Publication SHALL be derived exclusively from an Interpretation Result." (GLOSSARY.md §3)

### Ontological Category

**Publication is an OBJECT.**

It is a discrete, identifiable entity that:
- Has content (the information message)
- Has origin (derived from Interpretation Result)
- Has destination (distributed through channels)
- Has lifecycle (generated, published, archived)

### Analysis

Publication is the ATOMIC UNIT of information in the system.

It is the smallest indivisible object that can be distributed to residents.

### Conclusion

Publication is an OBJECT — the atomic unit of information.

---

## 2.3 Publication Package

### Evidence

> "Publication Package: The complete collection of Publications generated during one Reporting Period." (GLOSSARY.md §3)

> "A Publication Package MAY contain: planned outage publications; emergency outage publications; tomorrow publications; graphical publications; service announcements." (GLOSSARY.md §3)

### Ontological Category

**Publication Package is a COLLECTION.**

It is a grouping of Publications based on:
- Time (one Reporting Period)
- Content (various publication types)

### Analysis

Publication Package is a TEMPORAL COLLECTION that aggregates Publications for one Reporting Period.

It is not an atomic object — it is a container.

### Conclusion

Publication Package is a COLLECTION — a temporal grouping of Publications.

---

## 2.4 Reporting Period

### Evidence

> "Reporting Period: The time interval for which Publications are generated." (GLOSSARY.md §2)

> "A Reporting Period MAY correspond to: a calendar day; an update cycle; another officially defined reporting interval." (GLOSSARY.md §2)

### Ontological Category

**Reporting Period is an IDENTITY DIMENSION.**

It is not an object or a collection — it is a dimension that identifies WHEN Publications belong together.

### Analysis

Reporting Period is a TEMPORAL BOUNDARY that defines:
- When a set of Publications begins
- When a set of Publications ends
- How Publications are grouped temporally

### Conclusion

Reporting Period is an IDENTITY DIMENSION — a temporal boundary.

---

## 2.5 Territory

### Evidence

> "Territory: A publication unit representing either: the Administrative Centre; one Starosta District." (GLOSSARY.md §4)

> "A Territory is a logical publication entity rather than an administrative classification." (GLOSSARY.md §4)

### Ontological Category

**Territory is an IDENTITY DIMENSION.**

It is not an object or a collection — it is a dimension that identifies WHERE Publications belong.

### Analysis

Territory is a SPATIAL BOUNDARY that defines:
- Which geographic area a Publication covers
- How Publications are grouped spatially
- Which residents a Publication serves

### Conclusion

Territory is an IDENTITY DIMENSION — a spatial boundary.

---

## 2.6 Issue

### Evidence

> "An Issue opens when the first Publication for a calendar day is generated. The Issue represents one editorial edition of the Journal for that day." (TELEGRAM_PUBLICATION_LIFECYCLE.md)

### Ontological Category

**Issue is a TEMPORAL INSTANTIATION.**

It is the concrete realization of the Journal for one day.

### Analysis

Issue is the DAILY INSTANCE of the Journal:
- It opens when the first Publication is generated
- It represents one editorial edition
- It closes when the day ends

### Conclusion

Issue is a TEMPORAL INSTANTIATION — the daily instance of the Journal.

---

## 2.7 Release

### Evidence

> User context: "One Journal Release may contain: Emergency outage publication, Planned outage publication..."

### Ontological Category

**Release is a COMPOSITE ENTITY.**

It is a collection of Publications organized by Territory for one Reporting Period.

### Analysis

Release is the COMPOSITE PRODUCT that:
- Contains multiple Publications
- Is organized by Territory
- Exists for one Reporting Period
- Is delivered to residents

### Conclusion

Release is a COMPOSITE ENTITY — the complete daily product.

---

## 2.8 Entry

### Evidence

No direct evidence in canonical documents.

### Ontological Category

**Entry is NOT defined in the project.**

It is a potential term for the atomic unit, but "Publication" already fills this role.

### Conclusion

Entry is NOT A CONCEPT in the current project ontology.

---

## 2.9 Notice

### Evidence

No direct evidence in canonical documents.

### Ontological Category

**Notice is NOT defined in the project.**

It is a potential term for a type of Publication, but "Publication" already covers this.

### Conclusion

Notice is NOT A CONCEPT in the current project ontology.

---

## 2.10 Information Object

### Evidence

No direct evidence in canonical documents.

### Ontological Category

**Information Object is NOT defined in the project.**

It is a generic term that could describe Publications, but "Publication" is more specific.

### Conclusion

Information Object is NOT A CONCEPT in the current project ontology.

# 7. Findings

## 7.1 Finding ONTO-001

**Journal is a polymorphic concept that cannot be reduced to a single ontological category.**

It is simultaneously a Service, Channel, and Archive.

**Evidence:** CHARTER.md §1, §10.

**Confidence:** HIGH.

## 7.2 Finding ONTO-002

**Publication is the atomic object — the smallest indivisible unit.**

It is the only concept that can be directly distributed to residents.

**Evidence:** GLOSSARY.md §3.

**Confidence:** HIGH.

## 7.3 Finding ONTO-003

**Reporting Period and Territory are identity dimensions, not objects.**

They identify WHEN and WHERE Publications belong, but are not Publications themselves.

**Evidence:** GLOSSARY.md §2, §4.

**Confidence:** HIGH.

## 7.4 Finding ONTO-004

**Issue is a temporal instantiation — the daily instance of the Journal.**

It opens when the first Publication is generated and represents one editorial edition.

**Evidence:** TELEGRAM_PUBLICATION_LIFECYCLE.md.

**Confidence:** HIGH.

## 7.5 Finding ONTO-005

**Release is a composite entity, but its existence is uncertain.**

It appears only in user context, not in canonical documents.

**Evidence:** User context only.

**Confidence:** MEDIUM.

## 7.6 Finding ONTO-006

**Entry, Notice, and Information Object are NOT concepts in the current project ontology.**

They have no evidence in canonical documents.

**Evidence:** Analysis of GLOSSARY.md, CHARTER.md.

**Confidence:** HIGH.
