# 1. Purpose

This document establishes the scope, context and research questions for CASE-JRN-001: reconstruction of the ontology of the SvitloSk Public Information Journal.

---

# 2. Case Scope

## 2.1 What This Case Investigates

The ontology of the SvitloSk Public Information Journal — what actually exists inside it, what is a representation, what is a process, what is an implementation detail.

## 2.2 What This Case Does NOT Investigate

- Publication channels (Telegram, Facebook, PWA)
- Software implementation
- Infrastructure
- Existing specification validity
- Previous investigation findings

## 2.3 Investigation Constraint

This investigation starts from zero.

It does NOT inherit terminology from CASE-PUB-001 or any previous investigation.

Everything must be proven from the project's mission and domain reality.

---

# 3. Context

## 3.1 Previous Investigation

CASE-PUB-001 identified "Journal Release" as the primary produced object.

The Architect is NOT satisfied.

There is insufficient proof that "Journal Release" is actually the correct primary object.

## 3.2 Recognition

Before designing any publication subsystem, the nature of the Public Information Journal itself must be reconstructed.

The Journal is the foundational concept. Everything else depends on it.

## 3.3 Implication

This investigation must determine what the Journal ACTUALLY is, not what previous investigations assumed it to be.

# 5. Research Questions

## 5.1 Q1: What is a Journal?

Not in publishing theory.
Not in software.
Inside THIS project.

## 5.2 Q2: Journal Ontological Category

Is Journal:
- Object
- Collection
- Process
- Identity
- Service
- Product
- Something else

Prove.

## 5.3 Q3: Journal Continuity

Does Journal exist continuously?
Or only during publication?

## 5.4 Q4: Journal Without Publications

Can Journal exist without publications?

## 5.5 Q5: Smallest Indivisible Object

What is the smallest indivisible object inside Journal?
- Publication?
- Entry?
- Message?
- Notice?
- Information Object?
- Other?

## 5.6 Q6: Issue Existence

Does Issue exist?
If yes — what is Issue?

## 5.7 Q7: Release Existence

Does Release exist?
If yes — what is Release?

## 5.8 Q8: Issue vs Release

Can Issue and Release both exist?
Or are they synonyms?

## 5.9 Q9: Release Contains Publications

Can one Release contain many Publications?

## 5.10 Q10: Publication Belongs to Releases

Can one Publication belong to several Releases?

## 5.11 Q11: Release Without Publisher

Can Release exist without Publisher?

## 5.12 Q12: Publisher Without Release

Can Publisher exist without Release?

## 5.13 Q13: Journal Without Publisher

Can Journal exist without Publisher?

## 5.14 Q14: Competing Models

Construct every possible ontology.
Minimum 5 competing models.

## 5.15 Q15: Model Evaluation

For every model evaluate:
- Simplicity
- Consistency
- Semantic completeness
- Compatibility with CHARTER
- Compatibility with previous investigations
- Contradictions

## 5.16 Q16: Hidden Assumptions

Find every hidden assumption.

## 5.17 Q17: Traditional Publishing Inheritance

Search for concepts unconsciously inherited from traditional publishing.
They may be wrong.

## 5.18 Q18: Ontological Categories

Determine whether Journal, Issue, Release, Publication, Entry, Notice, Information Object are different ontological categories or merely different names.

## 5.19 Q19: Dependency Graph

Construct the dependency graph.

## 5.20 Q20: Facts Only

Produce only FACTS.
Do not choose the best model.
