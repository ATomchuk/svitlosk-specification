# 3. Competing Models

## 3.1 Model A: Journal as Continuous Service

### Ontology

```
Journal (continuous service)
    ↓ produces
Publication (atomic unit)
    ↓ delivered via
Channel (Telegram, Facebook, PWA)
```

### Description

Journal is a CONTINUOUS SERVICE that produces Publications over time.

Journal exists independently of any specific Publication.

Journal is the ongoing process of analyzing, interpreting and distributing information.

### Identity

Journal has TEMPORAL identity (continuous existence).

Publication has EPHEMERAL identity (created, delivered, archived).

### Characteristics

- Journal is a SERVICE, not an OBJECT
- Journal exists continuously
- Publications are produced by the Journal
- Channels are delivery mechanisms

### Evaluation

| Criterion | Assessment |
|-----------|------------|
| Simplicity | HIGH — only two core concepts |
| Consistency | MEDIUM — "service" is unusual for ontological analysis |
| Semantic completeness | LOW — does not explain daily editions |
| CHARTER compatibility | HIGH — CHARTER says "public information service" |
| Previous investigation compatibility | LOW — does not match CASE-PUB-001 findings |

### Contradictions

- Does not explain how daily editions work
- Does not explain how information is organized by territory
- Does not explain temporary vs permanent publications

## 3.2 Model B: Journal as Daily Collection

### Ontology

```
Journal (ongoing stream)
    ↓ instantiated as
Daily Collection (one per day)
    ↓ contains
Publication (atomic unit)
    ↓ delivered via
Channel
```

### Description

Journal is an ONGOING STREAM that is instantiated as a Daily Collection each day.

Daily Collection contains all Publications for one day.

Publications are atomic units within the Daily Collection.

### Identity

Journal has CONTINUOUS identity.

Daily Collection has TEMPORAL identity (one per day).

Publication has CONTENT identity (what it says).

### Characteristics

- Journal is the abstract concept
- Daily Collection is the concrete instantiation
- Publications are organized within the Daily Collection
- Channels deliver Publications from the Daily Collection

### Evaluation

| Criterion | Assessment |
|-----------|------------|
| Simplicity | HIGH — three core concepts |
| Consistency | HIGH — logical hierarchy |
| Semantic completeness | MEDIUM — explains daily editions but not territory |
| CHARTER compatibility | HIGH — "daily publications" in CHARTER §10 |
| Previous investigation compatibility | MEDIUM — similar to CASE-PUB-001 "Journal Release" |

### Contradictions

- Does not explain territory organization
- Does not explain temporary vs permanent publications
- "Daily Collection" is a new term not in evidence

## 3.3 Model C: Journal as Territory-Organized Structure

### Ontology

```
Journal (ongoing stream)
    ↓ instantiated as
Daily Edition (one per day)
    ↓ organized by
Territory (Administrative Centre, Starosta Districts)
    ↓ contains
Publication (atomic unit)
    ↓ delivered via
Channel
```

### Description

Journal is an ONGOING STREAM that is instantiated as a Daily Edition each day.

Daily Edition is organized by Territory.

Each Territory has its own Publications.

### Identity

Journal has CONTINUOUS identity.

Daily Edition has TEMPORAL identity (one per day).

Territory has SPATIAL identity (geographic location).

Publication has CONTENT identity (what it says).

### Characteristics

- Journal is the abstract concept
- Daily Edition is the concrete instantiation
- Territory organizes the Daily Edition
- Publications are organized by Territory within the Daily Edition

### Evaluation

| Criterion | Assessment |
|-----------|------------|
| Simplicity | MEDIUM — four core concepts |
| Consistency | HIGH — matches territorial model |
| Semantic completeness | HIGH — explains daily editions and territory |
| CHARTER compatibility | HIGH — "daily publications" and community structure |
| Previous investigation compatibility | MEDIUM — extends CASE-PUB-001 with territory |

### Contradictions

- Does not explain temporary vs permanent publications
- Does not explain graph vs address publications
- "Daily Edition" is a new term not in evidence

## 3.4 Model D: Journal as Multi-Layer Structure

### Ontology

```
Journal (ongoing stream)
    ↓ contains
Layer: Emergency Information
Layer: Planned Information
Layer: Graphical Information
Layer: Technical Information
Layer: Service Information
    ↓ each layer contains
Publication (atomic unit)
    ↓ delivered via
Channel
```

### Description

Journal is an ONGOING STREAM that contains multiple LAYERS of information.

Each Layer represents a different type of information.

Publications are atomic units within each Layer.

### Identity

Journal has CONTINUOUS identity.

Layer has TYPE identity (what kind of information).

Publication has CONTENT identity (what it says).

### Characteristics

- Journal is the abstract concept
- Layers organize information by type
- Publications are organized by Layer
- Channels deliver Publications from Layers

### Evaluation

| Criterion | Assessment |
|-----------|------------|
| Simplicity | MEDIUM — three core concepts plus layers |
| Consistency | HIGH — matches user context (emergency, planned, graph, etc.) |
| Semantic completeness | HIGH — explains different publication types |
| CHARTER compatibility | MEDIUM — CHARTER does not mention layers |
| Previous investigation compatibility | LOW — does not match CASE-PUB-001 structure |

### Contradictions

- Does not explain territory organization
- Does not explain daily editions
- "Layer" is a new term not in evidence

## 3.5 Model E: Journal as Composite Identity

### Ontology

```
Journal (identity: time + territory + type)
    ↓ realized as
Information Object (atomic unit)
    ↓ delivered via
Channel
```

### Description

Journal is a COMPOSITE IDENTITY defined by three dimensions:
- Time (Reporting Period)
- Territory (geographic location)
- Type (emergency, planned, graphical, etc.)

Information Object is the atomic unit realized from this identity.

### Identity

Journal has COMPOSITE identity (time + territory + type).

Information Object has CONTENT identity (what it says).

### Characteristics

- Journal is the identity, not the object
- Information Object is the realization of the identity
- Channels deliver Information Objects
- The identity determines what exists

### Evaluation

| Criterion | Assessment |
|-----------|------------|
| Simplicity | LOW — complex identity model |
| Consistency | HIGH — logically rigorous |
| Semantic completeness | HIGH — explains all dimensions |
| CHARTER compatibility | MEDIUM — CHARTER does not mention composite identity |
| Previous investigation compatibility | LOW — fundamentally different approach |

### Contradictions

- Does not explain the relationship between identity and realization
- "Information Object" is a new term not in evidence
- May be over-engineered for the domain

## 3.6 Model F: Journal as Publication Container

### Ontology

```
Journal (container)
    ↓ contains
Publication (atomic unit)
    ↓ has attributes
    ├── Time (Reporting Period)
    ├── Territory (geographic location)
    ├── Type (emergency, planned, graphical, etc.)
    └── Status (temporary, permanent)
    ↓ delivered via
Channel
```

### Description

Journal is a CONTAINER that holds Publications.

Each Publication has attributes: Time, Territory, Type, Status.

Publications are atomic units within the Journal.

### Identity

Journal has CONTINUOUS identity.

Publication has ATTRIBUTE identity (time + territory + type + status).

### Characteristics

- Journal is the container
- Publications are the contents
- Attributes describe Publications
- Channels deliver Publications

### Evaluation

| Criterion | Assessment |
|-----------|------------|
| Simplicity | HIGH — two core concepts |
| Consistency | HIGH — straightforward containment |
| Semantic completeness | MEDIUM — explains attributes but not organization |
| CHARTER compatibility | HIGH — "public archive" and "daily publications" |
| Previous investigation compatibility | MEDIUM — similar to CASE-PUB-001 but simpler |

### Contradictions

- Does not explain how Publications are organized
- Does not explain daily editions
- "Container" is a technical term, not a domain term

# 5. Cross-Examination

## 5.1 Model A vs Model B

**Model A** treats Journal as a continuous service.

**Model B** treats Journal as an ongoing stream instantiated as Daily Collections.

**Contradiction:** Model A does not explain daily editions. Model B does.

**Resolution:** Model B is more complete than Model A.

## 5.2 Model B vs Model C

**Model B** organizes by day only.

**Model C** organizes by day AND territory.

**Contradiction:** Model B does not explain territory organization. Model C does.

**Resolution:** Model C is more complete than Model B.

## 5.3 Model C vs Model D

**Model C** organizes by territory.

**Model D** organizes by information type.

**Contradiction:** Model C does not explain information types. Model D does.

**Resolution:** These are complementary, not contradictory. A complete model may need both.

## 5.4 Model E vs All Others

**Model E** treats Journal as a composite identity.

**All others** treat Journal as a container or stream.

**Contradiction:** Model E is fundamentally different in ontological approach.

**Resolution:** Model E may be over-engineered for the domain.

## 5.5 Model F vs All Others

**Model F** treats Journal as a simple container.

**All others** have more structure.

**Contradiction:** Model F is simpler but less complete.

**Resolution:** Model F may be too simple for the domain.

# 6. Findings

## 6.1 Finding MODELS-001

**Six competing ontological models were constructed.**

Each model has different strengths and weaknesses.

**Evidence:** Analysis of CHARTER.md, GLOSSARY.md, ARCHITECTURE.md, User context.

**Confidence:** HIGH.

## 6.2 Finding MODELS-002

**Model C (Journal as Territory-Organized Structure) is most semantically complete.**

It explains daily editions, territory organization, and atomic publications.

**Evidence:** CHARTER.md §10, GLOSSARY.md §3, User context.

**Confidence:** HIGH.

## 6.3 Finding MODELS-003

**No single model is perfect.**

Each model has weaknesses that need to be addressed.

**Evidence:** Evaluation matrix above.

**Confidence:** HIGH.

## 6.4 Finding MODELS-004

**The Journal ontological category is unresolved.**

Models disagree on whether Journal is a service, stream, container, or identity.

**Evidence:** Model A (service), Model B (stream), Model F (container), Model E (identity).

**Confidence:** HIGH.
