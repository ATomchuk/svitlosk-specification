# 1. Purpose

This document analyzes whether the concepts Journal, Issue, Release, Publication, Entry, Notice, Information Object are different ontological categories or merely different names for the same concept.

---

# 2. Concept Analysis

## 2.1 Journal

### Definition

> "The SvitloSk Public Information Journal is the primary public communication channel of the project. It serves as a public archive of processed information, daily publications, analytical summaries and official project communications." (CHARTER.md §10)

### Ontological Category

**Journal is a POLYMORPHIC CONCEPT.**

It is simultaneously:
- A SERVICE (analyzes, interprets, distributes)
- A CHANNEL (delivers information)
- An ARCHIVE (stores historical information)

### Distinctiveness

Journal is DISTINCT from all other concepts because:
- It is the HIGHEST-LEVEL concept
- It encompasses all other concepts
- It has multiple functions (service, channel, archive)

---

## 2.2 Issue

### Definition

> "An Issue opens when the first Publication for a calendar day is generated. The Issue represents one editorial edition of the Journal for that day." (TELEGRAM_PUBLICATION_LIFECYCLE.md)

### Ontological Category

**Issue is a TEMPORAL INSTANTIATION.**

It is the DAILY INSTANCE of the Journal.

### Distinctiveness

Issue is DISTINCT from Journal because:
- Issue is TEMPORAL (one per day)
- Journal is CONTINUOUS (exists always)
- Issue is a CONCRETE instantiation of the abstract Journal

Issue is DISTINCT from Publication because:
- Issue CONTAINS Publications
- Publication is an ATOMIC unit
- Issue is a COLLECTION of Publications

---

## 2.3 Release

### Definition

> User context: "One Journal Release may contain: Emergency outage publication, Planned outage publication..."

### Ontological Category

**Release is a COMPOSITE ENTITY.**

It is a collection of Publications organized by Territory for one Reporting Period.

### Distinctiveness

Release is DISTINCT from Issue because:
- Release may be ORGANIZED BY TERRITORY
- Issue may not have explicit territory organization
- Release is a more detailed structure than Issue

Release is DISTINCT from Publication because:
- Release CONTAINS Publications
- Publication is an ATOMIC unit
- Release is a COLLECTION of Publications

### Uncertainty

Release is NOT defined in canonical documents.

Its existence is UNCERTAIN.

---

## 2.4 Publication

### Definition

> "Publication: A public information message generated by the SvitloSk system for distribution through one or more supported communication channels." (GLOSSARY.md §3)

### Ontological Category

**Publication is an OBJECT.**

It is the ATOMIC UNIT of information in the system.

### Distinctiveness

Publication is DISTINCT from all other concepts because:
- It is the SMALLEST INDIVISIBLE unit
- It can be DIRECTLY DISTRIBUTED to residents
- It is derived from an Interpretation Result

# 4. Synonym Analysis

## 4.1 Are Issue and Release Synonyms?

### Evidence

- Issue is defined in TELEGRAM_PUBLICATION_LIFECYCLE.md
- Release is used in user context
- Both refer to "one edition of the Journal for one day"

### Analysis

Issue and Release appear to describe the SAME CONCEPT:
- Both are daily instances of the Journal
- Both contain Publications
- Both exist for one Reporting Period

### Conclusion

Issue and Release are POTENTIAL SYNONYMS.

But Release may have ADDITIONAL characteristics (territory organization) that Issue does not.

### Status: UNCERTAIN

Requires further investigation.

## 4.2 Are Publication and Entry Synonyms?

### Evidence

- Publication is defined in GLOSSARY.md §3
- Entry is NOT defined anywhere

### Analysis

Publication is a CANONICAL concept.

### Conclusion

Publication and Entry are NOT synonyms.

Entry does not exist in the current ontology.

# 6. Findings

## 6.1 Finding TERM-001

**Journal, Issue, Publication, Publication Package, Reporting Period, and Territory are DISTINCT ontological categories.**

Each has a different category and different characteristics.

**Evidence:** CHARTER.md §10, GLOSSARY.md §2, §3, §4, TELEGRAM_PUBLICATION_LIFECYCLE.md.

**Confidence:** HIGH.

## 6.2 Finding TERM-002

**Issue and Release are POTENTIAL SYNONYMS.**

Both refer to "one edition of the Journal for one day."

But Release may have additional characteristics.

**Evidence:** TELEGRAM_PUBLICATION_LIFECYCLE.md, User context.

**Confidence:** MEDIUM.

## 6.3 Finding TERM-003

**Entry, Notice, and Information Object are NOT CONCEPTS in the current ontology.**

They have no definitions and no evidence.

**Evidence:** Analysis of GLOSSARY.md, CHARTER.md.

**Confidence:** HIGH.

## 6.4 Finding TERM-004

**The valid ontological categories are: Polymorphic (Journal), Temporal Instantiation (Issue), Object (Publication), Collection (Publication Package), Identity Dimension (Reporting Period, Territory).**

These are the distinct categories that exist in the current ontology.

**Evidence:** Analysis of all canonical documents.

**Confidence:** HIGH.
