# 2. Contradictions

## 2.1 CONTRADICTION-001: Single Parser vs Two Parsers

### Repository Statement

> "Parser: A software Component responsible for retrieving Open Data from approved Data Sources." (GLOSSARY.md §2)

> "Parser is the canonical architectural Component responsible for retrieving Open Data." (GLOSSARY.md §7)

**Implication:** Single Parser component.

### Architect Intent Statement

> "Parser A: Collect official Queue Schedule."

> "Parser B: Collect planned and emergency outages."

> "These parsers are independent."

**Implication:** Two independent Parser components.

### Analysis

Repository describes ONE Parser.

Architect Intent describes TWO independent Parsers.

### Severity: HIGH

This affects the architectural decomposition.

### Status: UNRESOLVED

Requires clarification.

## 2.6 CONTRADICTION-006: Address Lookup vs Domain Independence

### Repository Statement

> "Address Lookup: Allows resident to determine which subqueue serves their address." (Architect Intent)

**Implication:** Address Lookup bridges address-based and queue-based domains.

### Architect Intent Statement

> "Queue Schedule, Planned Outages, Emergency Outages are THREE DIFFERENT ONTOLOGICAL OBJECTS. Never merge them."

**Implication:** Domains must remain independent.

### Analysis

Address Lookup connects address information (Domains B/C) to queue information (Domain A).

This may violate domain independence.

### Severity: LOW

Address Lookup is a search function, not a domain merger.

### Status: RESOLVED

No contradiction — Address Lookup is a bridge, not a merger.
