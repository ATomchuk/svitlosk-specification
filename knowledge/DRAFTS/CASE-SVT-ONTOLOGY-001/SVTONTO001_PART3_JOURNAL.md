# 1. Purpose

This document determines the ontological status of Journal, Journal Edition, Publication, Release, and Issue, and their relationships.

---

# 2. Concept Analysis

## 2.1 Journal

### Repository Evidence

> "The SvitloSk Public Information Journal is the primary public communication channel of the project." (CHARTER.md §10)

> "It serves as a public archive of processed information, daily publications, analytical summaries and official project communications." (CHARTER.md §10)

### Architect Intent Evidence

> "The project also publishes a Public Information Journal."

> "One journal edition may contain: planned outages (today), emergency outages (today), planned outages (tomorrow), emergency outages (tomorrow), tomorrow queue graph, technical project messages, future project messages"

### Ontological Status

**Journal is an ONGOING SERVICE.**

It continuously publishes information about electricity supply.

It is NOT an object, NOT a collection, NOT a process.

It is a SERVICE that produces editions over time.

### Evidence

- CHARTER.md describes it as "primary public communication channel"
- Architect Intent describes it as something that "publishes"
- Journal exists continuously, not just during publication

---

## 2.2 Journal Edition

### Repository Evidence

No direct evidence in canonical documents.

### Architect Intent Evidence

> "One journal edition may contain: planned outages (today), emergency outages (today)..."

> "Journal publications are grouped by Starosta Districts."

### Ontological Status

**Journal Edition is a COMPOSITE OBJECT.**

It is one daily instance of the Journal.

It contains multiple Publications organized by Territory.

### Evidence

- Architect Intent explicitly mentions "journal edition"
- It contains multiple publication types
- It is organized by Starosta Districts

---

## 2.3 Publication

### Repository Evidence

> "Publication: A public information message generated by the SvitloSk system for distribution through one or more supported communication channels." (GLOSSARY.md §3)

### Architect Intent Evidence

> "Journal publications are grouped by Starosta Districts."

### Ontological Status

**Publication is an ATOMIC OBJECT.**

It is the smallest indivisible unit of information in the Journal.

It is one message for one Territory.

### Evidence

- GLOSSARY defines it as "a public information message"
- It is the unit that is distributed to channels
- It cannot be decomposed into smaller meaningful units

---

## 2.4 Release

### Repository Evidence

No definition in canonical documents.

### Architect Intent Evidence

No direct mention of "Release."

### Ontological Status

**Release is NOT DEFINED in either source.**

It appears only in previous investigations (CASE-PUB-001).

### Evidence

- Not in GLOSSARY
- Not in CHARTER

- Only in CASE-PUB-001 user context

---

## 2.5 Issue

### Repository Evidence

> "An Issue opens when the first Publication for a calendar day is generated. The Issue represents one editorial edition of the Journal for that day." (TELEGRAM_PUBLICATION_LIFECYCLE.md)

### Architect Intent Evidence

No direct mention of "Issue."

### Ontological Status

**Issue is a TEMPORAL INSTANTIATION.**

It is the daily instance of the Journal (similar to Journal Edition).

### Evidence

- TELEGRAM_PUBLICATION_LIFECYCLE.md defines it as "one editorial edition"
- It opens when first Publication is generated
- It represents one day of the Journal

# 3. Relationship Analysis

## 3.1 Journal vs Journal Edition

```
Journal (ongoing service)
    ↓ produces
Journal Edition (one per day)
```

**Relationship:** Journal produces Journal Editions.

Journal is the SERVICE.

Journal Edition is the PRODUCT.

## 3.2 Journal Edition vs Publication

```
Journal Edition (composite)
    ↓ contains
Publication (atomic, per Territory)
```

**Relationship:** Journal Edition contains Publications.

Journal Edition is the CONTAINER.

Publication is the CONTENT.

## 3.3 Journal Edition vs Issue

```
Journal Edition ≈ Issue
```

**Relationship:** Journal Edition and Issue are POTENTIAL SYNONYMS.

Both refer to "one edition of the Journal for one day."

But "Issue" is defined in Telegram context, while "Journal Edition" is channel-independent.

## 3.4 Journal Edition vs Release

```
Journal Edition ≈ Release?
```

**Relationship:** UNCERTAIN.

Release is not defined in either source.

It may be a synonym for Journal Edition.

## 3.5 Publication vs Territory

```
Publication → Territory (one per Territory)
```

**Relationship:** Publication is organized by Territory.

Each Territory has its own Publication.
