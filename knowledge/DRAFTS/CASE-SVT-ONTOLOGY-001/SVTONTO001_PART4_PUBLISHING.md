# 1. Purpose

This document determines the ontological status of Publishing Subsystem, Telegram Adapter, Facebook Adapter, Publication, and Representation.

---

# 2. Concept Analysis

## 2.1 Publishing Subsystem

### Repository Evidence

No direct evidence in canonical documents.

### Architect Intent Evidence

> "The project contains a Publishing Subsystem."

> "Its purpose is publication of the Journal."

> "NOT Telegram."

> "Architecture: Publishing Subsystem → Telegram Adapter / Facebook Adapter"

### Ontological Status

**Publishing Subsystem is a SERVICE.**

It is responsible for publishing the Journal through adapters.

### Evidence

- Architect Intent explicitly defines it as a subsystem
- Its purpose is publication of the Journal
- It contains adapters for different channels

---

## 2.2 Telegram Adapter

### Repository Evidence

> "Telegram Journal: The official public communication channel operated by the SvitloSk Project." (GLOSSARY.md §3)

### Architect Intent Evidence

> "Telegram Adapter"

> "Telegram and Facebook are PEER adapters."

> "Neither is primary."

### Ontological Status

**Telegram Adapter is a COMPONENT.**

It is a channel-specific implementation of the publishing interface.

### Evidence

- Architect Intent defines it as an adapter
- It is a peer to Facebook Adapter
- It handles Telegram-specific concerns

---

## 2.3 Facebook Adapter

### Repository Evidence

No direct evidence in canonical documents.

### Architect Intent Evidence

> "Facebook Adapter"

> "Telegram and Facebook are PEER adapters."

> "Neither is primary."

### Ontological Status

**Facebook Adapter is a COMPONENT.**

It is a channel-specific implementation of the publishing interface.

### Evidence

- Architect Intent defines it as an adapter
- It is a peer to Telegram Adapter
- It handles Facebook-specific concerns

---

## 2.4 Publication

### Repository Evidence

> "Publication: A public information message generated by the SvitloSk system for distribution through one or more supported communication channels." (GLOSSARY.md §3)

### Architect Intent Evidence

> "Journal publications are grouped by Starosta Districts."

### Ontological Status

**Publication is an ATOMIC OBJECT.**

It is the unit of information distributed through adapters.

### Evidence

- GLOSSARY defines it as "a public information message"
- It is the unit that adapters deliver
- It is channel-independent

---

## 2.5 Representation

### Repository Evidence

No direct definition in canonical documents.

### Architect Intent Evidence

No direct mention of "Representation."

### Ontological Status

**Representation is a CONCEPT, not a defined entity.**

It describes how a Publication is rendered for a specific channel.

### Evidence

- Not defined in GLOSSARY
- Not mentioned in Architect Intent
- Used in previous investigations (CASE-PUB-001)

# 4. Adapter Independence

## 4.1 Telegram Adapter Independence

Telegram Adapter is:
- Independent from Facebook Adapter
- Channel-specific (Telegram Bot API)
- Peer to Facebook Adapter
- NOT primary

## 4.2 Facebook Adapter Independence

Facebook Adapter is:
- Independent from Telegram Adapter
- Channel-specific (Facebook API)
- Peer to Telegram Adapter
- NOT primary

## 4.3 Future Adapters

Future adapters may exist:
- PWA Adapter (if needed)
- Other channel adapters
- All peers to existing adapters
