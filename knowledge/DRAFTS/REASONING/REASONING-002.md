**Title:** Why Identity ≠ Identifier

**Status:** Draft

---

# Reasoning Chain

## Step 1: Publication Has Identity Without Identifier

- Publication has Territory + Date + Template identity
- No specification defines a Publication ID
- DATA_MODEL.md requires "uniquely identifiable" but no ID exists
- This proves Identity and Identifier are separable

---

## Step 2: Telegram Message ID Is Assigned After Creation

- Publication exists in Generated state
- Generated state has no Telegram Message ID
- Telegram Message ID is assigned when Publication is delivered
- This proves Identifier is not Identity

---

## Step 3: Identity Persists, Identifier Can Change

- Publication identity (Territory + Date + Template) never changes
- Telegram Message ID can be released (when Publication is removed)
- If Publication is re-published, it gets a new Telegram Message ID
- This proves Identity persists while Identifier can change

---

## Step 4: Identity Is Intrinsic, Identifier Is Assigned

- Identity is WHAT the object IS (intrinsic property)
- Identifier is HOW the object is referred to (assigned label)
- Identity exists before any Identifier
- Identifier depends on external systems

---

## Step 5: This Distinction Affects Everything

- Ownership is determined by Identifier (Telegram Message ID)
- But Identity is what makes Publication the same over time
- Changing Identifier does not change Identity
- This distinction is fundamental to the entire identity model

---

# Conclusion

Identity and Identifier are fundamentally different concepts. Identity is intrinsic; Identifier is assigned. Identity persists; Identifier can change.
