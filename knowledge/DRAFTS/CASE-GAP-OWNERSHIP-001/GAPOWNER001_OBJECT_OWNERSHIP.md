# Object Gap Ownership

## GAP-OBJ-001: Publication Engine

**Owner:** Publisher

**Reason:** Publication Engine implements the Publisher Role. It is the canonical Component that realizes Publisher behavior.

**Decision Maker:** Publisher (decides what to generate)

**Executor:** Publication Engine (executes generation)

**Close Before Publisher Spec?** YES — this is a core Publisher concept.

**Move Outside Publisher?** NO — this IS Publisher.

**Evidence:** GLOSSARY §7, TELEGRAM_PUBLISHING_CANONICAL_MODEL.md §3.1

---

## GAP-OBJ-002: Schedule Generator

**Owner:** Parser (or separate component)

**Reason:** Schedule Generator transforms normalized data into schedules. This is a DATA PROCESSING function, not a publication function.

**Decision Maker:** Parser (determines when schedule is needed)

**Executor:** Schedule Generator (generates schedule)

**Close Before Publisher Spec?** NO — this is outside Publisher scope.

**Move Outside Publisher?** YES — this belongs to Parser or separate component.

**Evidence:** TELEGRAM_PUBLISHING_CANONICAL_MODEL.md §4.5
