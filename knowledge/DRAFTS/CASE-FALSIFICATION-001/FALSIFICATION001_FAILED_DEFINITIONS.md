# Failed Definitions

## CONCEPT

### Original Definition

An abstract idea that describes the domain.

### Failed Definition 1

An idea that describes the domain.

**Why Failed:** "Idea" is too broad. Could include concrete objects.

### Failed Definition 2

An abstract entity.

**Why Failed:** "Entity" implies existence, but concepts are abstract.

### Improved Definition

An abstract idea that describes the domain, cannot be created or destroyed, and is not directly observable.

---

## OBJECT

### Original Definition

A discrete, identifiable entity that exists in the system.

### Failed Definition 1

An entity that exists.

**Why Failed:** Too broad. Includes concepts.

### Failed Definition 2

A concrete thing.

**Why Failed:** "Thing" is too vague.

### Improved Definition

A discrete, identifiable entity that exists in the system, can be created and destroyed, and has state.

---

## INSTANCE

### Original Definition

A concrete realization of a type.

### Failed Definition 1

A realization.

**Why Failed:** "Realization" is too broad.

### Failed Definition 2

A specific example.

**Why Failed:** "Example" is too vague.

### Improved Definition

A concrete realization of a type that has identity and can be created and destroyed.

---

## COLLECTION

### Failed Definition 1

A group.

**Why Failed:** "Group" is too broad.

### Failed Definition 2

A set of things.

**Why Failed:** "Things" is too vague.

### Improved Definition

A group of related entities that has boundaries and can contain multiple entities.

## PROCESS

### Failed Definition 1

An activity.

**Why Failed:** "Activity" is too broad.

### Failed Definition 2

**Why Failed:** "Transformation" is too vague.

### Improved Definition

An activity.

**Why Failed:** "Activity" is too broad.

### Failed Definition 2

A capability.

### Improved Definition
