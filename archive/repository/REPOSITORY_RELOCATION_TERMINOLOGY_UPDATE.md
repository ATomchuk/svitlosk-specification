# Purpose

Establish consistent relocation terminology.

---

# 1. Approved Terminology

| Term | Definition | Usage |
|------|-----------|-------|
| Relocation | Moving a document to its architectural destination | When referring to all 198 root files |
| Archiving | Moving historical materials into archive/ | When referring specifically to archive/ |
| Migration | The complete process of reorganizing the repository | When referring to Stage R-5 as a whole |

---

# 2. Prohibited Terminology

| Prohibited Phrase | Correct Replacement |
|------------------|-------------------|
| "scheduled for archive" | "scheduled for relocation per approved Blueprint" |
| "moved to archive" | "relocated to architectural destination" |
| "archive-bound" | "scheduled for relocation" |
| "all root files go to archive" | "root files relocated per Blueprint" |

---

# 3. Distinction Rule

Relocation and Archiving SHALL NEVER be treated as synonyms.

Archiving is a SUBSET of relocation. Only historical materials are archived. Subsystem documents are relocated to their subsystem directories.
