# Architectural Research Process

## Phase 1: Research

**Purpose:** Investigate the subject from first principles.

**Steps:**

1. Define research question
2. Collect all repository evidence
3. Forget existing assumptions
4. Reconstruct from first principles
5. Build multiple competing models
6. Cross-examine every model
7. Perform collapse experiments
8. Analyze contradictions
9. Compare with external theory
10. Document all findings

**Output:** Research Report with findings, models, contradictions

**Duration:** Variable (days to weeks)

**Participants:** CLI (researcher)

---

## Phase 2: Alternative Models

**Purpose:** Present multiple competing models without choosing.

**Steps:**

1. Construct 5+ independent models
2. For each model: definition, evidence, strengths, weaknesses
3. Cross-examine every model
4. Identify contradictions between models
5. Do NOT choose between models
6. Document all models

**Output:** Model Inventory with competing models

**Duration:** Variable

**Participants:** CLI (researcher)

---

## Phase 3: Hearing

**Purpose:** Adversarial examination of all models.

**Steps:**

1. Present every model
2. Attack every model with evidence
3. Defend every model against attacks
4. Identify unresolved contradictions
5. Do NOT resolve contradictions
6. Document all attacks and defenses

**Output:** Hearing Record with attacks, defenses, contradictions

**Duration:** Variable

**Participants:** CLI (researcher)

---

## Phase 4: Architect Decision

**Purpose:** Architect makes informed decision based on evidence.

**Steps:**

1. Review Research Report
2. Review Model Inventory
3. Review Hearing Record
4. Consider all evidence
5. Make decision
6. Document decision with reasoning

**Output:** Architect Decision with reasoning

**Duration:** Variable

**Participants:** Architect (decision-maker)

---

## Phase 5: Knowledge Extraction

**Purpose:** Extract validated findings from research.

**Steps:**

1. Identify findings confirmed by multiple investigations
2. Create knowledge findings with evidence
3. Assign confidence levels
4. Document repository impact
5. Create extraction backlog
6. Migrate to canonical knowledge

**Output:** Knowledge Findings with evidence

**Duration:** Variable

**Participants:** CLI (researcher), Architect (approval)

---

## Phase 6: Canonical Knowledge

**Purpose:** Integrate validated findings into repository.

**Steps:**

1. Review Knowledge Findings
2. Architect approves findings
3. Create knowledge files in repository
4. Update INDEX.md
5. Verify repository consistency

**Output:** Canonical Knowledge in repository

**Duration:** Variable

**Participants:** Architect (approval), CLI (implementation)

---

## Phase 7: Repository

**Purpose:** Permanent storage of architectural knowledge.

**Steps:**

1. Store Knowledge Findings in knowledge/findings/
2. Store Principles in knowledge/principles/
3. Update knowledge/INDEX.md
4. Verify repository structure
5. Archive investigation artifacts

**Output:** Repository with preserved knowledge

**Duration:** Variable

**Participants:** CLI (implementation)

# Key Insight

**The process is LINEAR — each phase depends on the previous.**

**The process is MANDATORY — all 7 phases must be completed.**

**The process is SEPARATED — research and decision are distinct phases.**

**The process is PERMANENT — it applies to every future investigation.**
