# Reconstructed Identity Model

**Identity is WHAT an object IS — its essential nature that makes it "the same object" throughout its lifecycle.**

**Key properties:**

- Created at object creation
- Persists across all lifecycle states
- Survives Identifier changes
- Survives Representation changes
- Cannot be destroyed (only the object can be destroyed)
- Cannot be duplicated (one object = one Identity)

---

# Reconstructed Identifier Model

**Identifier is HOW an object is REFERRED TO — a label, key, or reference used to point to the object.**

**Key properties:**

- Assigned after Identity exists (for most objects)
- May change between Representations
- May be platform-dependent
- May be released when object is deleted
- Multiple Identifiers can reference one Identity
- One Identifier should reference one Identity
