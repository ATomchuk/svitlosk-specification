# Purpose

This document investigates whether Publication, Telegram Message, Publication Package, Repository File, Graphic, Text, Schedule, Historical Record are Objects or Representations.

---

# Classification

## Publication

**Classification:** OBJECT

**Evidence:**

- GLOSSARY.md §3: "A public information message generated by the SvitloSk system"
- Has intrinsic Identity (Territory + Date + Template)
- Has lifecycle (Generated → Published → Updated → Archived → Removed)
- Is the canonical concept

**Reason:** Publication is the primary domain object.

---

## Telegram Message

**Classification:** REPRESENTATION

**Evidence:**

- TJS-000A §10: "The platform identifier assigned to a published message"
- Borrows Identity from Publication
- Exists only after Publication is delivered
- Is platform-specific

**Reason:** Telegram Message is a projection of Publication on Telegram.

---

## Publication Package

**Classification:** OBJECT (Collection)

**Evidence:**

- GLOSSARY.md §3: "The complete collection of Publications generated during one Reporting Period"
- Contains Publications
- Has no independent content
- Is a business concept

**Reason:** Publication Package is a collection object, not a representation.

---

## Repository File

**Classification:** REPRESENTATION

**Evidence:**

- TJS-021 §3.1: "Repository SHALL be the single authoritative source of truth"
- Borrows Identity from Publication
- Is a storage mechanism
- Is implementation-specific

**Reason:** Repository File is a projection of Publication in storage.

---

## Graphic

**Classification:** OBJECT

**Evidence:**

- TJS-022: "Graphic publications transform normalized outage data into visual formats"
- Has intrinsic Identity (unique per generation)
- Has lifecycle (Generated → Published → Archived)
- Is a domain concept

**Reason:** Graphic is a domain object representing visual information.

---

## Text

**Classification:** REPRESENTATION

**Evidence:**

- TJS-005: "Publication Grammar" defines text structure
- Is a format, not a concept
- Is derived from Publication
- Is implementation-specific

**Reason:** Text is a format used to represent Publication content.

---

## Schedule

**Classification:** OBJECT

**Evidence:**

- GLOSSARY.md §5: "The official schedule of planned electricity availability"
- Has intrinsic Identity (unique per generation)
- Has lifecycle (Generated → Published → Archived)
- Is a domain concept

**Reason:** Schedule is a domain object representing electricity availability.

---

## Historical Record

**Classification:** REPRESENTATION

**Evidence:**

- GLOSSARY.md §2: "The persistent storage of historical Interpretation Results and Publications"
- Borrows Identity from Publication
- Is an archive mechanism
- Is permanent

**Reason:** Historical Record is a projection of Publication in archive.

---

# Classification Summary

| Item | Classification | Reason |
|------|---------------|--------|
| Publication | OBJECT | Primary domain object |
| Telegram Message | REPRESENTATION | Platform projection |
| Publication Package | OBJECT (Collection) | Business collection |
| Repository File | REPRESENTATION | Storage projection |
| Graphic | OBJECT | Domain object |
| Text | REPRESENTATION | Format representation |
| Schedule | OBJECT | Domain object |
| Historical Record | REPRESENTATION | Archive projection |

---

# Key Insight

**Domain concepts (Publication, Graphic, Schedule) are OBJECTS.**

**Projections (Telegram Message, Repository File, Historical Record) are REPRESENTATIONS.**

**Collections (Publication Package) are OBJECTS.**

**Formats (Text) are REPRESENTATIONS.**
