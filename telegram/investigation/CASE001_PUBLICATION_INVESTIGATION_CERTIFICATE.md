# Inconsistencies Found

## Inconsistency I-001: Definition Emphasis

GLOSSARY emphasizes "message" and "distribution."

TELEGRAM_GLOSSARY emphasizes "unit" and "belonging to Issue."

DATA_MODEL emphasizes "representation" and "public distribution."

**Status:** UNRESOLVED

## Inconsistency I-002: Lifecycle States

Legacy TJS-002 includes "Scheduled" state.

Current TJS-021 does not include "Scheduled" state.

**Status:** UNRESOLVED

## Inconsistency I-003: Ownership Model

TJS-010: Publication Engine owns Publications.

Legacy TJS-007: Publisher owns Publications.

No document addresses Manual Publication ownership.

**Status:** UNRESOLVED

## Inconsistency I-004: Platform Independence

TJS-000: Journal exists independently from any platform.

But all lifecycle mechanics reference Telegram.

Telegram Message ID is part of Publication identity.

**Status:** UNRESOLVED

## Inconsistency I-005: Updated State

TJS-021 defines Updated as a lifecycle state.

Updated always transitions back to Published.

No other transition from Updated exists.

**Status:** UNRESOLVED

## Inconsistency I-006: Archival Trigger

TJS-021: "when the reporting period ends"

Legacy TJS-008: "next day approximately 05:00"

No precise definition of "reporting period end."

**Status:** UNRESOLVED

## Inconsistency I-007: Identity Criteria

No explicit identity criteria defined.

Possible candidates: Territory + Date, Territory + Template, Telegram Message ID.

**Status:** UNRESOLVED

## Inconsistency I-008: Graphic Publication Lifecycle

TJS-022 defines Graphic Publications.

TJS-022 does not define lifecycle for Graphic Publications.

**Status:** UNRESOLVED

## Inconsistency I-009: Circular Dependency

TJS-000: Publication belongs to Issue.

TJS-021: Issue opens when first Publication is generated.

Circular dependency exists.

**Status:** UNRESOLVED
