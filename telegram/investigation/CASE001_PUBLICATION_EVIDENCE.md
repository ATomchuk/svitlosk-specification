# Evidence E-005: TJS-010 — Publishing Model Description

**Location:** TJS-010 §3.1, §4.1, §10

**Description:**

- Publication Engine transforms normalized dataset into deterministic Publication Package
- Publication is generated from normalized data
- Publication is distributed through Telegram Publisher to Telegram Channel
- Data flow: Parser → Normalized Dataset → Publication Engine → Publication Package → Telegram Publisher → Telegram Channel → Subscribers

**Source level:** Normative Telegram Specification

---

# Evidence E-006: TJS-021 — Publication Lifecycle Definition

**Location:** TJS-021 §4, §5, §6

**Lifecycle states:**

1. Generated — created from normalized data, not yet published
2. Published — live and visible to readers
3. Updated — modified after initial publication
4. Archived — preserved as historical record
5. Removed — deleted (temporary Publications only)

**Source level:** Normative Telegram Specification

---
