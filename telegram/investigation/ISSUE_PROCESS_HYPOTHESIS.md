# Purpose

This document investigates Hypothesis H2: Issue is NOT an Object — it is a Process.

---

# Hypothesis H2: Issue is a Process

**Definition:** Issue is a sequence of actions that transforms inputs into outputs, bounded by start and end triggers.

---

# Supporting Evidence

## Evidence 1: Issue Has Start Trigger

**Source:** TJS-021 §8.1

**Quote:** "An Issue opens when the first Publication for a calendar day is generated"

**Analysis:** Issue starts when an external event occurs (first Publication generated). This is characteristic of a process, not an object.

---

## Evidence 2: Issue Has Activities

**Source:** TJS-021 §8.2

**Quote:** "An Issue is maintained through: Publication creation, updates, replacement, removal"

**Analysis:** Issue is defined by activities (creation, updates, replacement, removal), not by static properties.

---

## Evidence 3: Issue Has End Trigger

**Source:** TJS-021 §8.3

**Quote:** "An Issue closes when: All Publications for the day have been archived, no further updates are expected, the next day's Issue opens"

**Analysis:** Issue ends when external conditions are met, not when an internal property changes.

---

## Evidence 4: Issue Is Defined by Actions

**Source:** TJS-021 §8

**Analysis:** Issue lifecycle is described as actions (opens, maintained, closes), not as states (exists, doesn't exist).

---

## Evidence 5: Issue Depends on External Trigger

**Source:** TJS-021 §8.1

**Analysis:** Issue cannot start without an external trigger (first Publication). Objects typically start when created, not when triggered.

---

## Evidence 6: Issue Is Bounded by Activity

**Source:** TJS-021 §8

**Analysis:** Issue exists only while editorial activity is happening. When activity stops (all Publications archived), Issue closes.

---

# Contradicting Evidence

## Contradiction 1: Issue Persists After Process Ends

**Source:** TJS-021 §8.3

**Quote:** "The Issue remains available as part of the historical record after closing"

**Analysis:** Processes typically end and disappear. Issue persists as historical record.

---

## Contradiction 2: Issue Has Identity

**Source:** TJS-000 §4

**Quote:** "Issue represents one editorial edition... for a single calendar day"

**Analysis:** Processes typically don't have identity — they have start/end times. Issue has identity (Calendar day).

---

## Contradiction 3: Issue Is Treated as Thing

**Source:** TJS-000 §4 (hierarchy)

**Analysis:** Issue is positioned as a thing in the semantic hierarchy, not as a process.

---

## Contradiction 4: Issue Has Properties

**Source:** TJS-000 §5

**Quote:** "Issue — Owns: Daily edition, date scope"

**Analysis:** Processes typically don't "own" things. Issue owns daily edition and date scope.

---

# Verdict

**H2 is PARTIALLY SUPPORTED.**

Issue exhibits Process properties (start trigger, activities, end trigger, action-based definition, external dependency, activity-bounded) but contradicts others (persists after end, has identity, treated as thing, has properties).
